## Phase 3: AI triage pipeline — audited-pass
Notes: `lib/ai/emergencyKeywords.ts` — regex word-boundary keyword fast-path (gas, smoke, spark*, co, carbon monoxide, burning). `lib/ai/schema.ts` — zod schema for LLM triage output. `lib/ai/anthropicClient.ts` — calls claude-haiku via tool-use for structured output. `lib/ai/triage.ts` — `runTriage()`: keyword flag always computed; merges with LLM result (`isEmergency = keywordFlag || llm.isEmergency`); no API key → pending fallback (classification "pending", serviceNotes = raw input verbatim); malformed LLM output (zod parse fail) or thrown error → same pending fallback, keyword flag preserved. Wired into `app/actions/submitLead.ts` after lead+"submitted" log: runs triage, `lib/repo/triageResult.ts` stores result, logs "triaged" ActionLog; the triage/store/log sequence is wrapped in try/catch so any failure (including `createTriageResult` throwing) logs "triage_failed" instead of throwing past the point the Lead is already saved — lead intake always succeeds (rule 11). Eval set `lib/ai/evals/cases.ts` (12 cases incl. the vague 9pm gas-smell story from PLAN.md) + `evals.test.ts` — zero missed emergencies on gas/CO/smoke/spark/burning cases, zero false positives on routine cases, run via `npm test` (deterministic, no live API key needed since keyword fast-path guarantees the result independent of the LLM). Bugs caught by tests during build: "co" substring-matched "cold" (fixed with word boundaries), "sparks" keyword didn't match "sparking" (fixed with `\bspark\w*` stem match).
Audit: 2026-07-07 — first pass audited-fail: `app/actions/submitLead.ts` had no try/catch around the triage/store/log sequence — if `createTriageResult` threw (e.g. DB error), the whole server action would throw after the Lead was already committed, breaking rule 11 ("intake must never fail because triage failed") and leaving no ActionLog trace of the failure. Fixed: wrapped the sequence in try/catch, catch branch logs "triage_failed" ActionLog and still returns `success: true`. Added `app/actions/submitLead.triageFailure.test.ts` (mocks `createTriageResult` to throw) proving the lead is created, "submitted" + "triage_failed" logs exist, and `submitLead` never throws. Re-verified: `npx vitest run` 9 files/29 passed, `npx tsc --noEmit` clean, `npx eslint .` clean. All Phase 3 Done-when criteria and RULES.md items 6, 7, 9, 10, 11, 12, 14 confirmed.
